# Known Limitations

1. Cross-role messages, professional lead handling, developer inventory changes, agency invitations, and admin case actions use illustrative client-side responses rather than durable authenticated APIs.
2. Cloud uploads, private documents, signed media URLs, malware scanning, CDN delivery, and large-file behavior are not implemented end to end.
3. Email, WhatsApp, SMS, push, payment processing, and provider webhooks are not configured.
4. Background workers, retry queues, production error monitoring, alerting, and backup restoration have not completed pilot verification.
5. Saved homes, booking requests, and inspection requests are durable; other catalogue and community preview activity is not fully synchronized across devices.
6. The accessibility review includes automated semantics and responsive visual evidence; it is not a full WCAG conformance audit with assistive-technology users.
7. Property inventory, workspace people, analytics, businesses, ratings, and conversations shown in the public catalogue are illustrative and labelled accordingly.
8. The shared demo credential is appropriate only for controlled demonstrations and must be rotated before broader access.
